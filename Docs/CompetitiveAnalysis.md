### 2.4 iOS 18 Built-in Flashlight ⚡ (Стратегический конкурент)

Наиболее значимый конкурент — **встроенный** в iOS функционал.

**Ключевые изменения iOS 18 (2024):**
- Новый UI: кривая-индикатор вместо вертикальных линий
- **Dual-axis control:** вертикаль = яркость, горизонталь = ширина луча
- **Beam width** управляет массивом 3×3 LED (центральные + периферийные)
- Хаптик-отклик на взаимодействие
- Живой preview эффекта на экране

**Ограничение:** Управление шириной луча — только iPhone 14 Pro / 15 Pro / 16 Pro (Adaptive True Tone flash).

**Что iOS НЕ умеет:**
- Строб
- Текст → Морзе (свет или звук)
- Звуковой вывод
- Экспорт аудио
- Обучение Морзе

**Итог:** iOS 18 закрывает базовый flashlight-сценарий для большинства пользователей, но **не закрывает** нишу Морзе.

**Полный список функций:**

- Текст → звук (синтез аудио)
- **Создание аудио-файлов** Морзе для экспорта/рингтонов
- Декодирование Морзе с микрофона в реальном времени (с автокалибровкой)

**Дизайн:**
- Профессиональный, функционально перегруженный
- Ориентирован на опытных пользователей (ham radio)
- Большая кривая обучения для новичков

**Слабости:**
- Сложный UX — не для casual-пользователей
- Фокус на обучении, а не на быстрой передаче
- Subscription vibe отпугивает пользователей

## 4. Дизайн-паттерны

### 4.1 Flashlight-приложения (эмпирические нормы)

```
┌─────────────────────┐
│   [тёмный фон]       │  ← ОБЯЗАТЕЛЬНО: работа в темноте
│                      │
│    ◉  OFF/ON         │  ← Большая кнопка центр, thumb-zone
│                      │
│  ████████░░  75%     │  ← Слайдер яркости горизонтальный
│                      │
│  ──Строб──  SOS──    │  ← Вторичные режимы мелкими кнопками
└─────────────────────┘
```

**Правила:**
- Фонарик должен включаться за ≤1 касание от запуска приложения
- Слайдер яркости — горизонтальный (большой палец удобнее)
- Ни одного модального окна при старте (no onboarding splash)
- Лог-шкала яркости ощущается лучше, чем линейная

### 4.2 Morse-приложения (паттерны)

```
┌─────────────────────┐
│  ┌─────────────────┐│
│  │ Введите текст…  ││  ← TextField вверху
│  └─────────────────┘│
│  ... --- ...         │  ← Превью моноширинным шрифтом
│  ─────────────────── │
│  [ ▶ Свет ] [▶Звук] │  ← Два CTA рядом
│  [ Поделиться ]      │
│  ─────────────────── │
│  ████░ скорость      │  ← Слайдер скорости внизу
└─────────────────────┘
```

**Правила:**
- Моноширинный шрифт для `.` и `-` (читаемость кода)
- Stop-кнопка появляется ТОЛЬКО во время передачи
- Визуальный/анимированный индикатор активной передачи

### 4.3 iOS 18 Benchmark — что задаёт стандарт

Apple ввела **dual-axis interactive control** для фонарика:
- Drag вверх/вниз = яркость
- Drag влево/вправо = ширина луча
- Хаптик-фидбек на каждое изменение
- Живая анимация луча отражает реальный физический эффект

**Вывод для MorseLight:** Наш brightness slider не хуже; но можно добавить хаптик-отклик при включении/выключении.

### 4.4 Permission UX (best practices 2024–2025)

**Антипаттерн (распространён у конкурентов):**
- Системный диалог разрешения вылетает сразу при запуске без контекста
- Результат: 60-70% пользователей отказывают

**Best practice (используем в MorseLight):**
1. **Pre-permission priming:** показать экран с объяснением ДО системного диалога
2. **Contextual request:** запрашивать разрешение только в момент использования функции
3. **Status display:** показывать текущий статус разрешений в UI (наш PermissionRow)
4. **Recovery UX:** при отказе — показывать кнопку "Открыть настройки"

## 5. Пользовательские Pain Points

Собрано из App Store отзывов и форумов:

### 5.1 Топ жалоб по всей нише

| # | Проблема | Источник | Частота |
|---|---------|---------|---------|
| 1 | **Реклама прерывает передачу** — случайный клик на баннер гасит фонарик | Multiple apps | Очень часто |
| 2 | **Подписка за базовые функции** — $2.99/мес за конвертер текста воспринимается как грабёж | Morse Code R&D отзывы | Часто |
| 3 | **"Rave mode"** — случайная активация строба без понятного способа выключить | iTorch-подобные | Регулярно |
| 4 | **Фонарик включается на максимуме при старте** — вместо последнего настроенного уровня | Несколько приложений | Часто |
| 5 | **Крэш при вводе текста** — в ряде Morse-приложений | Отзывы пользователей | Умеренно |
| 6 | **Нет отображения Морзе-строки** — не видно что будет передано | Большинство простых app | Умеренно |
| 7 | **Только фиксированная скорость** — нет регулировки WPM | Базовые Morse apps | Умеренно |
| 8 | **Нет совместного использования аудио** — нельзя отправить звуковой файл | Почти все конкуренты | Редко (нишевый запрос) |

### 5.2 Что пользователи хвалят

- «Моментальный запуск» (ключевой фактор для фонаря)
- «Просто: кнопка, диммер, выкл» — одобрение минимализма
- Реальное применение строба/SOS в ЧС-ситуациях
- Создание аудио-файлов Морзе для позывных/рингтонов

## 6. Монетизация

### 6.1 Модели рынка

| Модель | Примеры | Средняя выручка | Риски |
|--------|---------|----------------|-------|
| **Free + Ads** | Flashlight+, iTorch (некоторые) | Низкая, стабильная | Прерывает UX, жалобы |
| **Free + IAP One-Time** | Morse-It ($4.99) | Средняя | Плато роста |
| **Free + Subscription** | Morse Code R&D ($2.99/мес) | Высокая (если удержать) | Отток, рейтинг ↓ |
| **Полностью бесплатно** | MorseKey, open source | $0 | — |
| **Разовая покупка (paid)** | MorseTorch Pro | Стабильная | Нет recurrent revenue |

### 6.2 Стратегия для MorseLight

На старте — **полностью бесплатно без рекламы**:
- Дифференциация от ad-supported конкурентов
- Быстрый рост рейтинга (no ad complaints)
- После набора аудитории — опциональный IAP "Pro" ($1.99–$2.99):
  - Morse decoder (камера + микрофон)
  - Waveform visualizer при воспроизведении
  - Siri Shortcuts интеграция
  - Кастомные тоны (частота, форма волны)

## 7. Gap Analysis: что делают конкуренты и чего нет у MorseLight

### 7.1 Функциональные пробелы (приоритизированы)

| Приоритет | Фича | Конкуренты с ней | Сложность |
|-----------|------|-----------------|-----------|
| 🔴 Высокий | **Мигание экраном** как альтернатива фонарику (iPad, sim) | Morse-It, Morse R&D | Низкая |
| 🔴 Высокий | **Вибрация** Морзе | Morse Code R&D | Низкая |
| 🟡 Средний | **Декодирование Морзе** (ввод точек/тире) | Morse-It, Morster | Средняя |
| 🟡 Средний | **Siri Shortcuts** (Create Morse audio from text) | Morse-It | Средняя |
| 🟡 Средний | **WPM-дисплей** вместо raw unitDuration | Morse-It, Morse R&D | Низкая |
| 🟡 Средний | **Control Center виджет** (iOS 18 API) | Нет у конкурентов | Средняя |
| 🟢 Низкий | **Строб** с регулировкой | iTorch | Низкая |
| 🟢 Низкий | **Хаптик-фидбек** при вкл/выкл | iOS 18 native | Низкая |
| 🟢 Низкий | **Декод с камеры** (visual Morse) | MorseTorch (HackGT) | Высокая |
| 🟢 Низкий | **Обучение Морзе** (Koch trainer) | Morse-It | Высокая |

### 7.2 Уникальные преимущества MorseLight (текущие)

1. **Экспорт WAV-аудио** — только Morse-It имеет это, и там за Premium
2. **Секция разрешений с UI** — никто не показывает статус Camera/Mic/Photos в отдельном разделе
3. **Строгая архитектура без рекламы** — clean experience от старта
4. **Живой превью Морзе** обновляется при каждом вводе символа
5. **Слайдер скорости** (unitDuration) — у многих конкурентов скорость фиксирована
6. **Swift 6 strict concurrency + actor** — надёжность передачи без гонок
